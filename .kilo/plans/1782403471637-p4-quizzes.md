# P4 ‚Äî Quizzes

Generate a mixed quiz (MCQ / flashcard / AI-graded short answer) from a single chat,
take it, get graded, and review attempts ‚Äî in both browser and desktop runtimes.

Spec: `refinement/architecture.md` ¬ß6-7, `refinement/phased-plan.md` P4. P3 (labs) is
the structural template; quizzes add an **attempt/grading lifecycle**.

## Locked decisions

1. **Generation** ‚Äî single structured-output call producing the whole mixed quiz,
   prompt-driven fenced-JSON (mirrors `generateLab`). Retry ‚â§2 on parse failure feeding
   the bad output back + a corrective instruction. Model decides the mix (~6-10 questions
   across mcq/flashcard/short). Editable `quizPrompt` settings override (parity with `labPrompt`).
2. **No "save raw" affordance** for quizzes (unlike labs) ‚Äî raw quiz text is not useful.
   On parse failure: surface `QuizGenerationError` as an error banner with a **Regenerate** button.
3. **Grading is immediate per-question.** MCQ + flashcard are auto-scored locally (no provider call);
   flashcard = **self-graded flip** (reveal back ‚Üí "Got it"/"Missed", recorded as `isCorrect`).
4. **Short-answer AI grading** via a renamed, **pure** method that takes data in (not a questionId):
   `gradeShortAnswer({ prompt, rubric, answer, context })` ‚Üí `{ isCorrect, feedback }`.
   `context` = `assembleContext(quiz.chatId)` so the grader is grounded in the source chat.
   Prompt-driven fenced-JSON; retry ‚â§2.
5. **Attempts: retakes allowed.** Each "Retake" starts a fresh attempt (new answers). Attempt
   history view. **Score = integer correct-count** (`isCorrect === true`); attempt `finishedAt`
   set when every question is answered.
6. **Short-answer grading failure** (exhausts retries / transport error): leave that answer
   `isCorrect = null` (ungraded ‚Üí doesn't count against the score), store an error message in
   `aiFeedback`, and show a **Re-grade** button on that question.

## Interface change (the P1 stubs become real)

`src/lib/ai/types.ts`:

```ts
// generateQuiz: Promise<never>  ->  Promise<GeneratedQuiz>
generateQuiz(messages: ChatMessage[], opts?: ChatStreamOptions): Promise<GeneratedQuiz>;

// gradeAnswer(questionId, answer): Promise<never>  ->
//   gradeShortAnswer(opts): Promise<GradedAnswer>   (pure: data in, data out)
gradeShortAnswer(opts: {
	prompt: string;        // the question prompt
	rubric: string;        // from the short payload
	answer: string;        // user's answer
	context: ChatMessage[]; // assembled source chat (grounding)
}): Promise<GradedAnswer>;
```

Re-export `GeneratedQuiz` and `GradedAnswer` from `types.ts` (same pattern as `GeneratedLab`).
The old `gradeAnswer(questionId, answer)` signature is dropped ‚Äî it could never reach the rubric.

## Data flow

### Generate (`quizzesStore.generate(chatId)`)
`assembleContext(chatId)` ‚Äñ `getActiveProvider()` ‚Üí `provider.generateQuiz(ctx)` ‚Üí
`toQuizQuestions(generated)` (assign uuid + ord) ‚Üí `quizzesRepo.create` + loop
`quizQuestionsRepo.add` ‚Üí return id (navigate `/quiz/<id>`). Errors: `AbortError` swallowed;
`QuizGenerationError` ‚Üí error banner + Regenerate; transport ‚Üí `formatProviderError`.

### Take / grade (`quizzesStore`)
- `startAttempt()` ‚Üí `quizAttemptsRepo.start`.
- Per question:
  - **mcq** ‚Üí on select: `record(answer=selectedIndex)` then `grade(isCorrect = idx===answerIndex)`.
  - **flashcard** ‚Üí reveal back, then self-mark: `record(answer=mark)` then `grade(isCorrect = mark)`.
  - **short** ‚Üí on submit: `record(answer)` then `gradeShortAnswer(...)` ‚Üí `grade({isCorrect, aiFeedback})`.
- When all answered ‚Üí recompute score (count `isCorrect===true`) ‚Üí `quizAttemptsRepo.finish(id, score)`.
- `retake()` ‚Üí `startAttempt()` again (fresh answers). `history` = `quizAttemptsRepo.listByQuiz`.

## Validation (acceptance)

Automated: `pnpm test` (new suites), `pnpm check` (svelte-check), `pnpm lint`. All must pass.
Manual:
- **Browser:** `pnpm dev` ‚Üí open a chat ‚Üí "Generate quiz" ‚Üí questions stream in ‚Üí take it ‚Üí MCQ
  scores instantly, flashcard flips + self-marks, short answer gets AI feedback inline ‚Üí summary
  shows score ‚Üí Retake starts a fresh attempt ‚Üí **reload** persists quiz + attempt + answers.
- **Desktop:** `pnpm tauri dev` ‚Üí same flow survives an **app restart**.
- **Provider switch:** add a second provider, set active, generate+grade again.
- **Grading failure:** with an offline/empty provider, a short answer stays ungraded with a Re-grade affordance.

## Risks / notes

- Quiz JSON is larger than a lab payload (options arrays). Single-call is still fine at ~6-10 questions;
  keep `MAX_ATTEMPTS` bounded (reuse 3). If generation consistently overflows context later, split is a
  future seam ‚Äî not now.
- `stubs.ts` deletion: grep for any remaining `p3`/`p4`/`quizStub`/`gradeAnswerP4` references before removing.
- Keep WASM DB off the main thread where latency matters (existing driver pattern); no new long queries introduced.
