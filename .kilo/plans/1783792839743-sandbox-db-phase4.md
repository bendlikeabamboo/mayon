## Goal

The sidecar hosts an isolated SQLite instance that MCP tools / sandboxed compute can use,
exposed to the web app via the `StorageDriver` seam (`SidecarDriver`) and a browsable,
read-write inspector under `/settings`. OPFS remains the app's sole primary store; this
DB is **not** wired as primary and never holds app data.

## Resolved decisions (from this planning pass)

1. **MCP-consumer scope = plumbing + surfaced path (no MCP template).** Ship the SQL
   endpoint, `SidecarDriver`, and inspector. Surface the DB path so a user can paste it
   into a *custom* stdio MCP server's args. We do **not** ship a "Sandbox SQLite" MCP
   template — the canonical `@modelcontextprotocol/server-sqlite` was removed from the
   official MCP repo, so a built-in template would be fragile. The DoD's "configured MCP
   server" is satisfied by a user-added custom stdio server pointing at the surfaced path.
2. **Inspector query box = read-write.** Any SQL (CREATE/INSERT/UPDATE/DROP) is accepted.
   Justified: it is isolated sandbox data, and power users must seed tables for MCP tools
   to query. Labelled clearly as a sandbox.
3. **DB path is surfaced via `HealthResponse`, not a new endpoint.** Add optional
   `sandboxDbPath?: string` to `HealthResponse` (present only with the `sandbox-db` cap);
   `sidecarStatus` carries it. The inspector reads from the store — single discovery contract.
4. **Wire protocol = one op-discriminated `POST /api/db/query`.** Request is
   `{ op: 'query'|'batch'|'exec', ... }`; response is the natural JSON for each op.
   Query/batch results include **both** `columns: string[]` and `rows: unknown[][]` so the
   inspector can render headers, while `SidecarDriver` maps to the positional
   `StorageDriver` contract (dropping `columns`).
5. **`SidecarDriver` implements `query/batch/exec` only.** `snapshot/restore/dispose` are
   omitted (optional on the seam); P5's file-level backup endpoints own persistence.
6. **The inspector does NOT go through `SidecarDriver`.** It calls `/api/db/query` via a
   thin `sandbox-db` client to retain `columns`. `SidecarDriver` exists to satisfy the
   `StorageDriver` contract (and prove `createDb(driver)` could target it); the inspector
   is a separate, repo-isolated power tool (per parent plan P4.2).
7. **Single long-lived better-sqlite3 connection**, opened at sidecar boot in WAL +
   `busy_timeout=5000`. Synchronous API → no pool needed.

## Grounding seams (reuse, don't reinvent)

- `StorageDriver` contract — `src/lib/db/driver/types.ts:17` (`query/batch/exec` +
  optional `snapshot/restore/dispose`). `BatchStatement` at `:12`.
- Reference driver impl — `src/lib/db/driver/memory.ts:38` (array-mode rows, BEGIN/COMMIT/
  ROLLBACK batch). Mirror its semantics.
- drizzle proxy — `src/lib/db/driver/proxy.ts:12` (`createDb(driver)`); proving it round-
  trips against `SidecarDriver` is the strongest contract test.
- Sidecar route pattern — `sidecar/src/llm-proxy.ts:13` (`registerLlmProxy(app)`); mirror
  for `registerSandboxDb(app, db)`. Test pattern — `sidecar/src/mcp.test.ts` (buildApp on
  port 0, `app.inject`).
- Sidecar HTTP client — `src/lib/sidecar/client.ts:2` (`sidecarClient.http(path, init)`).
- Status store — `src/lib/sidecar/status.svelte.ts:9` (`markConnected`) + `has(cap)` `:23`.
- Settings section stack — `src/routes/settings/+page.svelte:10` (render new section
  inside the `<ProviderConfig>` wrapper, after `<DataSection />`).
- MCP env model — `src/lib/mcp/types.ts:57` (`env: Record<string,{secretRef}>` is
  **secret-only**). Non-secret config like a DB path goes in `args` (cf. Filesystem
  template `templates.ts:35` passing `/tmp`). This is why the surfaced path is pasted into
  a custom server's args, not auto-injected as env.

## Hard rules (from AGENTS.md — non-negotiable)

- Components/stores call repositories only — **except** the sanctioned dev/power-tool
  exception the parent plan grants the inspector (it reaches the driver/endpoint directly,
  never `repos`, never app `db`). Keep it visibly isolated.
- No secrets in `settings`. The sandbox DB holds no secrets (no key tables); confirm the
  inspector never reads app tables (it only ever points at `/data/sandbox.sqlite`).
- Sidecar binds only to the internal docker network; nginx is the single same-origin entry.
  Never expose the raw sidecar port.

## Tasks

### P4.1 — Shared wire types + health extension
- `packages/shared/src/protocol.ts`:
  - `SidecarCap` already includes `'sandbox-db'` (`:1`) — no change there.
  - Add `sandboxDbPath?: string` to `HealthResponse` (`:2`).
  - Add:
    ```ts
    export interface DbBatchStatement { sql: string; params?: unknown[]; }
    export type DbQueryRequest =
      | { op: 'query'; sql: string; params?: unknown[] }
      | { op: 'batch'; stmts: DbBatchStatement[] }
      | { op: 'exec'; sql: string };
    export interface DbQueryResult { columns: string[]; rows: unknown[][]; }
    export type DbQueryResponse =
      | DbQueryResult                                   // op: query
      | { results: DbQueryResult[] }                    // op: batch
      | { changes: number; lastInsertRowid: number | bigint | null }; // op: exec
    export interface DbErrorResponse { error: string; detail?: string; }
    ```
  - Re-export from `packages/shared/src/index.ts` if it currently re-exports `protocol`.

### P4.2 — Sidecar SQL endpoint
- `sidecar/src/db.ts` (new):
  - `createSandboxDb(path = process.env.SANDBOX_DB_PATH ?? '/data/sandbox.sqlite')`:
    `new Database(path)`; `db.pragma('journal_mode = WAL')`; `db.pragma('busy_timeout = 5000')`.
    (WAL pragma is a harmless no-op on `:memory:`.)
  - `registerSandboxDb(app, db)` — `POST /api/db/query`:
    - Parse `DbQueryRequest`; 400 on malformed body / missing `op`.
    - `query`: `const stmt = db.prepare(sql); const rows = stmt.raw().all(...(params ?? []));
      const columns = stmt.columns().map(c => c.name);` → `{ columns, rows }`.
      (`raw()` yields positional arrays; `columns()` yields names — matches the contract.)
    - `batch`: run inside `db.transaction(() => …)`; map each stmt like `query`; on throw
      the transaction auto-rolls back (mirror `memory.ts:52` BEGIN/ROLLBACK semantics).
    - `exec`: `db.exec(sql)`; return `{ changes: db.changes, lastInsertRowid:
      db.lastInsertRowid }`.
    - Errors: any throw → `reply.code(400).send({ error, detail })` (SQLITE_BUSY etc.
      surface as 400 with the sqlite message; never 500 stack leak).
- `sidecar/src/server.ts`:
  - Import `createSandboxDb`, `registerSandboxDb`. In `buildApp()`, open the DB and
    register the route; **add `'sandbox-db'` to the caps array** and set
    `sandboxDbPath` on the health body (only present with the cap). Keep the
    "advertise only landed caps" invariant.
  - `db.ts` owns the connection lifecycle; `server.ts`'s `start()` may close it on
    shutdown (best-effort; fastify `onClose` hook).

### P4.4 — Web: `SidecarDriver` (StorageDriver contract)
- `src/lib/db/driver/sidecar.ts` (new) — `createSidecarDriver(): StorageDriver`:
  - `query<T>(sql, params = [])` → POST `{op:'query', sql, params}` via
    `sidecarClient.http('/api/db/query', …)`; on non-2xx throw `Error(body.error)`;
    return `{ rows: body.rows as T[] }` (positional; `columns` dropped).
  - `batch(stmts)` → POST `{op:'batch', stmts}` → `{ rows }` per result.
  - `exec(sql)` → POST `{op:'exec', sql}` → `void`.

  - Constructor does **not** gate on `sidecarStatus` (the driver is a dumb executor); the
    UI/callers gate. Throw a clear error if the endpoint is unreachable.

## Definition of Done

- **Sidecar suite:** `pnpm --filter @mayon/sidecar test` green — db endpoint (query
  columns+rows, batch transaction rollback, exec changes, 400 error mapping) + updated
  health caps assertion.
- **Root suite:** `pnpm lint && pnpm check && pnpm test` green — `SidecarDriver` contract
  + drizzle round-trip proof, inspector client, fake-fetch error paths.
- **Manual, browser + sidecar:** `docker compose up` → header shows sidecar connected with
  `sandbox-db` in caps → `/settings` **Sandbox DB** section appears, showing the path
  `/data/sandbox.sqlite` → run `CREATE TABLE notes(id INTEGER PRIMARY KEY, t TEXT)` then
  `INSERT INTO notes(t) VALUES('hi')` in the query box → `SELECT * FROM notes` renders a
  table with headers `id`/`t` and the row → add a **custom stdio** MCP server whose args
  point at the copied `/data/sandbox.sqlite` path → it reads the same `notes` row →
  `docker compose restart sidecar` → data persists (volume). With the sidecar down, the
  section is hidden and the rest of the app is unaffected.
- **No regression:** the P0–P3 browser gates (DB ready, theme persists, provider stream,
  HTTP MCP, Anthropic via proxy) pass unchanged.

## Risks

- **better-sqlite3 native build on `node:22-alpine` (musl).** v12 ships musl prebuilds in
  most cases, but if `docker compose build` fails to link, add `apk add --no-cache
  python3 make g++` to the sidecar `Dockerfile` **build** stage only. Validate via
  `docker compose build && docker compose up` as a first step.
- **Two processes, one file.** The sidecar's own connection **and** a spawned MCP server
  subprocess both open `/data/sandbox.sqlite`. WAL + `busy_timeout=5000` serializes
  writers and prevents corruption; under heavy concurrent writes a rare `SQLITE_BUSY`
  (after the timeout) is possible and surfaces as a clear 400. Accepted for a sandbox.
- **Cap-gating drift.** `sandbox-db` must be advertised **only** when the route is
  registered (it always is now). The `server.test.ts` exact-array assertion is the guard;
  updating it (P4.3) is mandatory, not optional.
- **Inspector never touches app data.** It only ever reaches `/data/sandbox.sqlite` via
  the sidecar; it has no handle to the OPFS DB. Keep it out of `repos`/`db` imports.

## Out of scope (explicit)

- Wiring `SidecarDriver` as the app's primary store (OPFS stays sole primary).
- `snapshot`/`restore` on `SidecarDriver` (P5 file-level backup owns this).
- A built-in "Sandbox SQLite" MCP template (deferred; decoupled from package churn).
- Cross-DB queries or attaching the OPFS DB to the sandbox.
- Multi-tenant / per-session sandbox DBs (single shared `sandbox.sqlite`).
