# AG1 — Port onto the Vercel AI SDK (no behavior change)

**Source of truth:** `refinement/agentic-capabilities.md` (design, decisions #1–12
locked) and `refinement/agentic-capabilities-phased.md` §AG1. Treat those as
authoritative; this is the implementation-ready breakdown grounded in the
shipped code.

## Goal

Swap the AI engine beneath everything — the four hand-rolled adapters, the
SSE/NDJSON stream parsers, and the fenced-JSON generation protocol — for the
Vercel AI SDK (`streamText` / `generateObject`), with **byte-for-byte today's
UX**. The single thing Mayon contributes is a per-provider custom-`fetch` shim
that preserves the P5 guarantee (no API key in the webview on desktop). This is
the spike that de-risks the whole agency posture (design §2.3, §4.2, §10 Phase A).

**No schema migration, no `db:generate` / `bundle:migrations`** in this phase.

## Resolved decisions (for this phase)

| # | Decision | Resolution |
| - | -------- | ---------- |
| 1 | Gate block | **Keep inline.** The gate is a *render-time* extraction from the streamed chat reply (not a `generateObject` call). `generate-gate.ts` + `extractFencedBlock` display usage are **preserved**; only the *generation* fence parsers retire. Converting the gate to a `generateObject` tail is a behavior change → deferred to a later phase. |
| 2 | `Provider` interface | **Retired.** `getActiveProvider()` returns `{ model, config }` where `model` is an SDK `LanguageModel`. Stores/orchestrators take the SDK model directly (matches AG3's loop calling `streamText` raw). No facade. |
| 3 | Desktop error status | **JS-first.** The desktop fetch shim wraps the existing Rust bridge events into a `Response` (`Headers`→status, `Chunk`→body, `Error`→`Response{ status, body: message }`). `transport.rs` is touched **only** if verification shows the SDK can't read the error body it needs (it can — today's desktop error body is already just `p.message`). |
| 4 | SDK version | **Stable major, not beta** (decision #12), confirmed in the opening spike (task 1). `ollama-ai-provider-v2` for Ollama. `zod` already in `package.json`. |
| 5 | Deletion sequencing | **Staged within the phase.** Land the new SDK path alongside the old code first (add + port), switch the stores, verify green tests + manual acceptance, **then** delete retired files. Old adapters/parsers stay importable until the port is verified. |

## Ordered task list

### Task 1 — Spike: pin the SDK major
- Add to `package.json` deps: `ai`, `@ai-sdk/openai`, `@ai-sdk/anthropic`,
  `@ai-sdk/google`, `ollama-ai-provider-v2` (stable majors; do **not** add betas).
  **Remove nothing yet.**
- Verify: (a) bundles cleanly under `@sveltejs/adapter-static` (no SSR / no Node
  builtins leaking into the SPA bundle); (b) the pinned `ai` major exposes the
  test utilities the ported tests need (`MockLanguageModel` / mocked stream
  helpers) and the `custom fetch` / `providerOptions` extension points.
- If a provider package lags the pinned `ai` major, pin that provider to a
  compatible minor and note it (decision #12: reversible).

### Task 2 — Per-provider keychain fetch shim
**New file: `src/lib/ai/sdk-fetch.ts`**

A `createKeychainFetch(auth)` returning the SDK's custom `fetch`, built **per
provider instance** capturing `keyId` + auth scheme. This is the direct
translation of today's `StreamInit.auth` descriptor (`transport.ts:27`) onto the
SDK's documented custom-`fetch` extension point.

- **Browser path** — read the key from `BrowserKeyStore.get(keyId)` → set
  `auth.header` (scheme-prefixed) → `fetch(url, init)`. Errors map through the
  existing `classifyFetchError` (TypeError→`CorsBlockedError` cross-origin /
  `NetworkError` otherwise) and `httpStatusToError` (429→`RateLimitError`,
  else `ProviderHttpError`). **No threat-model change** vs today's
  `createFetchTransport` (`http-transport.ts:36`).
- **Desktop path** — do **not** read the key. Emit a `keyInjection` descriptor
  `{ header, scheme, keyId }` and delegate to the existing Rust `llm_stream`
  bridge (`tauri-transport.ts`). Wrap the bridge's byte stream into a `Response`:
  status from the `Headers` event, body from `Chunk` events, and on `Error` build
  a `new Response(message, { status })` so the SDK throws `AI_APICallError` from
  a real status (not an in-band stream error). The `tauri-transport.ts`
  `HttpStreamTransport` backbone survives as the shim's backbone (design §4.2).
- Reuse the existing `getHttpTransport()` seam where it simplifies the desktop
  branch; the shim owns the `Response` construction the SDK requires.

### Task 3 — SDK factory + error mapping
**New files: `src/lib/ai/sdk-factory.ts`, `src/lib/ai/sdk-errors.ts`**

- `sdk-factory.ts` — `buildSdkModel(config, { hasKey }): LanguageModel` per
  `ProviderKind`, wiring task 2's fetch:
  - `openai-compatible` (OpenAI, Z.AI, OpenRouter, Kilo Gateway) → `@ai-sdk/openai`
    `createOpenAI({ baseURL: config.baseUrl, fetch, apiKey? })`. Browser passes
    `apiKey`-equivalent via the shim header; desktop omits it (key resolves in Rust).
  - `anthropic` → `@ai-sdk/anthropic`; `gemini` → `@ai-sdk/google`;
    `ollama` → `ollama-ai-provider-v2`.
- **Reasoning-mode parity (highest-fidelity risk).** Map `ReasoningMode`
  (`auto`/`enabled`/`disabled`) onto SDK `providerOptions` **per provider**:
  Anthropic → `thinking` budget (mirrors `anthropic.ts:69`); Gemini →
  `thinkingConfig.thinkingBudget` (mirrors `gemini.ts:78`); **Z.AI/GLM `thinking`
  field** — not a native `@ai-sdk/openai` field; pass through as an OpenAI
  `providerOptions`/raw-body extension that reproduces today's
  `thinkingField()` (`openai-compatible.ts:109`). Verify byte parity against
  today's Z.AI request body.
- `sdk-errors.ts` — `mapSdkError(err)` at the call boundary: SDK
  `AI_APICallError` → `RateLimitError` (429) / `CorsBlockedError` (browser
  TypeError on cross-origin) / `ProviderHttpError` (other), so
  `formatProviderError` (`errors.ts`) keeps working unchanged. One mapping.
- **Modify `client.ts`** — `getActiveProvider()` returns `{ model, config }`
  via `buildSdkModel`; `listProviders` / `saveProviders` / key accessors /
  `kindRequiresKey` / `discoverProviderModels` unchanged (the key lives in the
  runtime `KeyStore`, untouched). `MissingKeyError` still thrown fast pre-network.

### Task 4 — Port generation orchestrators → `generateObject` + Zod
**Modify:** `src/lib/ai/generate/{generate,generate-quiz,generate-brief,generate-title}.ts`

- Each becomes `(model, messages, opts)` using `generateObject({ model, schema,
  system, messages, abortSignal })` (lab/quiz/brief/grade) or
  `generateText` (title). **Reuse the existing Zod schemas unchanged** as the
  `generateObject` schemas: `GeneratedLabSchema` (`lab.ts`), `GeneratedQuizSchema`
  + `GradedAnswerSchema` (`quiz.ts`), `GeneratedBriefSchema` (`generate-brief.ts`).
- **Drop the `accumulate` retry loops** (`generate.ts:172`, `generate-quiz.ts:328`,
  `generate-brief.ts:102`) — the SDK retries internally (`maxRetries`).
- **Preserve the settings-overridable prompt KVs** (`labPrompt`, `quizPrompt`,
  `briefPrompt`) — they pass through as the `system` instruction.
- **Preserve the `*GenerationError` "raw" affordance** (`LabGenerationError` /
  `QuizGenerationError` / `BriefGenerationError` / `GradeError`): on SDK final
  failure, wrap into the existing typed error carrying the last raw output so the
  labs/quizzes store's "save raw anyway" path is unchanged.
- Title: `generateText` → `cleanTitle` (no JSON, no fence). Always `reasoning:
  'disabled'` (mirrors `generate-title.ts:39`).
- The **lab/quiz/brief *shapes* are unchanged**; the parser is now the SDK + Zod.
- **Gate (`generate-gate.ts`) is NOT touched** — it has no orchestrator and is
  render-time only (task 1 decision #1).

### Task 5 — Port `chatStore.send` → `streamText`; update stores
- **`chat.svelte.ts`** — `send` calls `streamText({ model, messages: ctx,
  abortSignal, providerOptions })` and accumulates from `result.textStream`
  (`this.streamBuffer += delta`). **Preserve verbatim:** the parallel
  `autoTitleRoot` / `inferBriefRoot` (with their separate `AbortController`s)
  and the DEV `strategy-lint` hook (`chat.svelte.ts:196`). Partial-text-on-error
  persistence contract unchanged.
- **`labs.svelte.ts`** — `provider.generateLab(ctx, …)` → `generateLab(model,
  ctx, …)`.
- **`quizzes.svelte.ts`** — `provider.generateQuiz(…)` → `generateQuiz(model, …)`;
  `provider.gradeShortAnswer(…)` → `gradeShortAnswer(model, …)`.

### Task 8 — Acceptance
- `pnpm test` / `pnpm check` / `pnpm lint` clean.
- **Browser:** `pnpm dev` → `/settings` provider flow + `/chat` streaming behave
  exactly as today (incl. the DEV `DbStatus` self-check, parallel title/brief
  inference, and the CORS "use desktop" fallback for Anthropic).
- **Desktop:** `pnpm tauri dev` → the key stays in the OS keychain (DevTools

  including Anthropic (no `dangerous-direct-browser-access`).
- **Artifact parity:** generate a lab, generate + take + grade a quiz, infer a
  brief, set a title — byte-for-byte today's behavior on the new engine.
- **Reasoning-mode parity:** toggle reasoning enabled/disabled on Z.AI, Anthropic,
  and Gemini and confirm today's per-provider behavior (esp. the Z.AI `thinking`
  field) is preserved.

## Risks / edge cases
- **Z.AI `thinking` field passthrough.** `@ai-sdk/openai` has no native Z.AI
  `thinking` field; reproducing `thinkingField()` (`openai-compatible.ts:109`) may
  require a `providerOptions`/raw-body extension. This is the single highest
  behavior-parity risk — verify it explicitly in task 8.
- **generateObject schema parity.** Lab/quiz/brief Zod schemas must accept exactly
  today's payloads (nested code-fence escaping, bare-string checklist coercion).
  Pin the ported tests to *resolved shapes*, not wire text (design AG1 risk).
- **`*GenerationError` raw affordance.** The labs/quizzes "save raw anyway" path
  depends on the typed generation errors carrying `raw`. With `generateObject`,
  raw text isn't naturally available — reconstruct it (the SDK exposes the failed
  response) so the affordance survives.
- **Rust bridge error-body fidelity.** JS-first `Response` construction (task 2)
  carries `p.message` as the body (today's behavior). Only escalate to a
  `transport.rs` change if verification shows the SDK needs the provider's raw
  error JSON (design AG1 risk; flagged "spike, may be needed").
- **SDK bundle size** in the `adapter-static` SPA. Confirm in task 1 the provider
  packages don't bloat the browser bundle past acceptable.

- Gate → `generateObject` conversion (behavior change; deferred).
- Cross-chat agency / generative tools (AG4–AG6).
