## Locked decisions (from planning session)

1. **`sidecar-data` volume → `server-data`.** Accept that the sandbox
   `sandbox.sqlite` is orphaned on upgrade (sandbox = MCP-tool scratch data, not
   app data; PG data lives in the separate `pg-data` volume, unaffected). Document
   a one-line upgrade note in the new docs.
2. **Docs: full rewrite.** Bring every stale doc in line with server-required +
   PG-primary. **Remove all** Tauri/desktop/OPFS/"browser-only"/"no server
   needed"/`bundle:migrations` references entirely (no "Tauri planned" stub).
3. **`DbRuntime` collapse** to `'pg' | 'unknown'`. Drop dead variants
   `'tauri' | 'browser' | 'memory'`.
4. **`translatePlaceholders`: hard-delete** (no shim). Port the 3 remaining
   raw-`?` emitters to `$1`; any future accidental `?` becomes a loud PG syntax
   error (fail-fast, desired).
5. From the master plan (unchallenged): delete OPFS + memory drivers; remove
   `@sqlite.org/sqlite-wasm`, `sql.js`, `@types/sql.js`; remove COEP/COOP plugin.
   **Keep** `better-sqlite3` (sandbox DB, D11) and `@electric-sql/pglite` (tests).
