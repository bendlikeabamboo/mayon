# Lean DB tests — fix `pnpm test` OOM (8 GB → <2 GB, zero coverage loss)

## Context & root cause

`pnpm test` (root, runs only `src/`) frequently OOM-kills in WSL (~8 GB). The cause
is **not** that the tests are inherently heavy — it is a lifecycle bug in how the
in-process Postgres (**PGlite**, a WASM Postgres, ~50–150 MB resident per instance)
is used:

1. **Leak — never disposed.** Every DB-backed test calls `bootstrapTestDb()`
   (`src/lib/db/driver/pg-test.ts`) which runs `new PGlite()`. The driver exposes
   `dispose()` → `client.close()` (`pg-test.ts:51`), but **none of the 18 `src/`
   test files ever call it**. Instances accumulate in the worker until it exits.
2. **Over-provisioned — fresh DB per *test case*, not per file.** Setup lives in
   `beforeEach`. `chat.svelte.test.ts` alone has **59 `it()` blocks** → 59 leaked
   PGlites + 59 full migration/FTS runs in one file (also `search.test.ts`=23,
   `repositories.test.ts`=19, `mcp.test.ts`=15, `quizzes`=15 …).
3. **Unbounded parallelism.** No `maxWorkers` cap in `vite.config.ts`, so many of
   these leaky files run at once.

Net: hundreds of unclosed WASM Postgres instances across parallel workers → 8 GB.

**Scope:** root `src/` tests only. The `server/` package tests already use a
disciplined `beforeAll`/`afterAll` shared PGlite pool (verified: `server/src/pg-import.test.ts`,
`pg-backup.test.ts`), so they are **out of scope**.

## Decision (confirmed)

Per-file shared PGlite + cheap per-test `TRUNCATE` reset + `dispose()` teardown +
worker cap. This is semantically identical to the current per-test fresh DB
(both give full per-test data isolation), so **no test logic or coverage changes**.
Side benefit: migrations/FTS run once per file instead of N× → tests also get faster.

## Changes

### 1. New lifecycle helper — `src/lib/db/driver/pg-test.ts`

Add a per-file factory (keep the existing `bootstrapTestDb` for any leftover ad-hoc
use, but migrate the suite to the new API):

```ts
import type { Db } from './proxy';

export interface FileTestDb { db: Db; driver: StorageDriver }

export function useFileTestDb(): {
  setup(): Promise<FileTestDb>;   // boot ONE PGlite: migrate + FTS bootstrap,
                                  // then bootstrapWithDriver(driver,'pg') so
                                  // getDb()/repos resolve (single code path)
  reset(): Promise<void>;         // TRUNCATE all public-schema tables
  teardown(): Promise<void>;      // driver.dispose?.()  (closes PGlite worker)
} {
  let handle: FileTestDb | null = null;
  return {
    async setup() {
      if (handle) return handle;
      const driver = createPgTestDriver();        // reuse existing internal factory
      await driver.init!();
      handle = { db: createDb(driver), driver };
      await bootstrapWithDriver(driver, 'pg');    // wire module-level getDb()/repos
      return handle;
    },
    async reset() {
      if (!handle) return;
      // dynamic + schema-resilient; RESTART IDENTITY resets sequences
      const res = await handle.driver.query(`
        SELECT string_agg(format('%I.%I', table_schema, table_name), ', ')
        FROM information_schema.tables
        WHERE table_schema = current_schema() AND table_type = 'BASE TABLE'`);
      const list = (res.rows[0] as unknown[])[0] as string | null;
      if (list) await handle.driver.exec(`TRUNCATE ${list} RESTART IDENTITY CASCADE`);
    },
    async teardown() {
      try { await handle?.driver.dispose?.(); } catch { /* best-effort */ }
      handle = null;
    }
  };
}
```

Notes:
- `createPgTestDriver` already exists in this file — reuse it; do not duplicate the
  migrate/FTS-bootstrap logic.
- Reset is **dynamic** (reads `information_schema`) so it survives future schema
  additions without editing a hardcoded table list. The 11 current tables:
  `chats, messages, branch_sources, cross_links, labs, quizzes, quiz_questions,
  quiz_attempts, quiz_answers, agent_traces, settings`.

### 2. Migrate all 18 DB-backed test files to the preamble pattern

Files (`rg -l bootstrapTestDb src/`, excluding `pg-test.ts` itself):

```
src/lib/mcp/{resources,mount,prompts}.test.ts
src/lib/chat/context.test.ts
src/lib/agent/{registry,deterministic-tools,generative-tools}.test.ts
src/lib/stores/{quizzes,chat,labs}.svelte.test.ts
src/lib/db/self-check.test.ts
src/lib/db/driver/{pg,proxy}.test.ts
src/lib/ai/keystore/migrate.test.ts
src/lib/db/repositories/{mcp,search,repositories}.test.ts
```

Each file gets the same preamble, replacing its `beforeEach(async () => { ... bootstrapTestDb() ... })`:

```ts
import { beforeAll, beforeEach, afterAll } from 'vitest';
import { useFileTestDb } from '$lib/db/driver/pg-test';

const testDb = useFileTestDb();
beforeAll(() => testDb.setup());     // boot once, wire getDb()/repos
beforeEach(() => testDb.reset());    // TRUNCATE ... CASCADE (runs FIRST)
afterAll(() => testDb.teardown());   // dispose()
```

Per-file notes:
- **Store/repo tests** (use `getDb()`/`repos`): delete the old `beforeEach`
  bootstrap line; the preamble's `setup()` wires the global. Keep any mock-reset
  `beforeEach`/`afterEach` — they run after `reset()` (registration order), which is
  the desired order.
- **Driver/proxy tests** (`pg.test.ts`, `proxy.test.ts`) that want the handle
  directly: read `testDb.db` / `testDb.driver` inside `it()` instead of calling
  `bootstrapTestDb()`.
- **migrate.test.ts**: keep its existing `beforeEach` that seeds legacy rows — it
  runs after the preamble `reset()`, so seeding starts from a clean table. Verified
  compatible (it needs clean data, not a fresh schema).
- **chat.svelte.test.ts** (nested `beforeEach` at line ~801): leave as-is; it only
  resets mocks.

### 3. Cap parallelism — `vite.config.ts`

Add to the `test` block a conservative, env-overridable worker cap (safety net on
top of the lifecycle fix):

```ts
test: {
  include: ['src/**/*.{test,spec}.{js,ts}'],
  environment: 'node',
  testTimeout: 15_000,
  minWorkers: 1,
  maxWorkers: Number(process.env.VITEST_MAX_WORKERS ?? 4)
}
```

(`maxWorkers` is also accepted by the root config; if the type insists on
`poolOptions`, use `poolOptions: { threads: { maxThreads: Number(...) } }`.)
Optional transition guard: `NODE_OPTIONS=--max-old-space-size=2048 pnpm test` to fail
fast rather than swap-thrash while validating.

## Risks & verification

- **Cross-test data dependency** (a test reading data another `it` wrote): currently
  impossible because each test already gets a fresh DB, so truncate-reset preserves
  semantics. Pre-check: `rg "beforeAll" src/**/*.test.ts` for any file that seeds
  in `beforeAll` and reads across `it`s — if found, move that seeding into a
  `beforeEach` (after `reset()`), or keep a local fresh DB for that file only.
- **PGlite `close()` reclamation**: with 1 instance per file this is moot, but
  confirm via RSS measurement that workers actually drop memory between files.
- **Identity/sequence reset**: `RESTART IDENTITY` resets sequences; no test should
  rely on a specific autoincrement value (none do — all use explicit IDs).

### Acceptance gates
- `pnpm lint && pnpm check && pnpm test` green; `pnpm --filter @mayon/server test`
  green (untouched, sanity only).
- **Memory:** measure peak RSS before/after, e.g.
  `/usr/bin/time -v pnpm test 2>&1 | rg "Maximum resident set size"`.
  Target: **< 2 GB** (realistically a few hundred MB). No WSL OOM-kill.
- **Speed:** wall-clock `pnpm test` should drop noticeably (migrations run
  1×/file instead of 1×/test).
- **Grep sweep:** `rg "bootstrapTestDb\(\)" src/**/*.test.ts` returns only
  intentional ad-hoc usages (ideally zero); all 18 files use `useFileTestDb()`.

## Out of scope
- `server/` package tests (already disciplined; `pnpm --filter @mayon/server test`).
- Dropping any DB test coverage (the leak was the problem, not the tests).
- Replacing PGlite with a shared external Postgres / testcontainers for `src/`
  (PGlite is fine when not leaked).
